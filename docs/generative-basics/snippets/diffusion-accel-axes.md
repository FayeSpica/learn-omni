# 扩散加速的两根轴：量化 vs 蒸馏（结论快照）

> 一个易混点：**LightX2V 的少步蒸馏 ≈ 量化吗?** 不是。两者是「推理加速」大类下**正交**的两根轴。本条把它们钉在一张表上，接回 NPU 侧的 [量化总览](../../vllm-omni/quantization-overview.md)。

## 一句话

一次扩散生成的开销 ≈ **步数 × 每步算力**。**量化砍「每步算力」，蒸馏砍「步数」**——改的东西、产物、能不能叠加，全都不同。

## 两根轴

| | **量化 quantization** | **步数蒸馏 step distillation** |
|---|---|---|
| 砍哪个因子 | 每步算力 / 显存 | 步数(50 → 4) |
| 手段 | 权重/激活 FP16 → INT8/INT4 | 训一个 LoRA，学会少步复现多步 |
| 步数变吗 | 不变 | 锐减 |
| 改的是 | 权重的**数值表示** | 模型的**行为/能力** |
| 产物 | 量化权重(表示替换) | distill **LoRA**(加法增量 `W+BA`) |
| 落地流程 | calibration / 量化算法 | 离线合并 LoRA → 装进 Diffusers 目录 |
| 谁的主场 | NPU/后端(昇腾量化) | 生成侧(LightX2V 等) |

**可叠加**:量化 + 4 步蒸馏一起上，加速最猛;两者不互斥。

## 归位

```text
推理加速
├── 量化      ── 砍"每步算力/显存"   ← NPU 侧主场
├── 蒸馏      ── 砍"步数"           ← LightX2V 在 PR #2134 用的这支
├── 缓存/复用 ── 砍"重复计算"(cache-dit 等)
└── 并行      ── TP / CFG-parallel，摊到多卡
```

## 一个必须记住的连带结论

少步蒸馏改的是「行为」，所以**采样器要跟着换**:蒸馏 LoRA 配 Euler 训练，推理就得用 Euler，用高阶的 UniPC 会崩(详见 [采样器篇](../samplers.md) 第四节)。量化则**不牵动采样器**——它只换数值表示，去噪循环的步进规则不变。这是两根轴"正交"最实际的体现。

!!! info "边界"
    LightX2V 是个视频加速**工具箱**，里面可能同时有蒸馏、量化、缓存;但 PR #2134 **只用了它的蒸馏支**(Distill-Loras + converter)，没用量化。vllm-omni 运行时不依赖 LightX2V，它只是权重/LoRA 的上游供应商。
