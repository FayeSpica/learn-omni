# 昇腾代次与原生低精度格式(A2 / A3 / A5·昇腾950)

> 速查:昇腾不同硬件代次(A2 = 910B / A3 = 910C / A5 = 昇腾950)各自**原生支持哪些低精度数据格式**(FP8 / MXFP8 / HiF8 / MXFP4 / INT8 …)。这是软件量化方案(W8A8/W4A8…)能否拿到原生算力的硬件底座。950 一列数据来自[《昇腾 950 NPU 架构白皮书》](https://public-download.obs.cn-east-2.myhuaweicloud.com/ascend/%E6%98%87%E8%85%BE950%20NPU%E6%9E%B6%E6%9E%84%E7%99%BD%E7%9A%AE%E4%B9%A6.pdf)(权威);A2/A3 两列为通用认知整理,落地以实测为准。

## 一、先分清两个层次

- **软件量化方案**:`W8A8 / W4A8 / W4A4 / W4A16 / KV C8` 等——由 ModelSlim 离线产出、vllm-ascend 的 `@register_scheme` 注册实现。见[量化特性支持速查](ascend-quantization.md)。
- **硬件原生数据格式**:`INT8 / FP8 / MXFP8 / HiF8 / MXFP4 …`——Cube/Vector 核能**直接计算**的精度。

量化方案最终要映射到硬件原生格式上。**同一份 vllm-ascend 代码,在不同代次硬件上跑同一个 MXFP4 权重,可能一个有原生算力、一个走降级/模拟路径**。代次差异的根在本篇这张表。

## 二、命名对齐

| 习惯叫法 | 芯片 | SoC 标识 | 状态 |
|---|---|---|---|
| **A2** | 910B 系列 | `ascend910b*` | 成熟主力 |
| **A3** | 910C / SuperPod | `ascend910_93` / `ascend910c` | 上一代旗舰 |
| **A5** | **昇腾 950**(950PR / 950DT) | — | 第三代达芬奇,本篇重点 |

> 昇腾 950 系列含两款共架构芯片:**950PR**(偏高性能推荐 / 大模型 Prefill,128GB·1.6TB/s)、**950DT**(偏大模型全量训练 + Decode/Prefill,144GB·4TB/s)。

## 三、原生低精度格式支持矩阵(按代次)

| 数据格式 | 位宽/编码 | A2(910B) | A3(910C) | **A5·昇腾950** |
|---|---|:--:|:--:|:--:|
| FP16 / BF16 | 16-bit | ✓ | ✓ | ✓ |
| TF32 | 19-bit(1-8-10) | ✓ | ✓ | ✓ |
| INT8 | 8-bit | ✓ | ✓ | ✓(Cube+Vector) |
| INT16 / 32 / 64 | Vector | 部分 | 部分 | ✓ |
| **FP8 E4M3** | 1-4-3 | ✗ | ✓ | ✓ |
| **FP8 E5M2** | 1-5-2 | ✗ | ✓ | ✓ |
| **MXFP8**(微缩 FP8) | FP8 + 8bit MX scale | ✗ | 有限 | ✓ **原生** |
| **HiF8**(华为自研) | 8-bit 变长前缀码 | ✗ | ✗ | ✓ **原生**(950 首发) |
| **MXFP4**(微缩 FP4) | FP4(1-2-1) + MX scale | ✗ | ✗ | ✓ **原生** |

> ✗ = 无原生硬件支持(可能不可用或走模拟/降级)。A2/A3 行为通用认知:**整数量化全代次成熟;FP8 从 A3 起;MXFP8/HiF8/MXFP4 是 950 才原生**。

### 关键结论

- **整数量化(INT8 系)**:三代都稳,A2 上最成熟。W8A8 / W4A8 / W8A16 / W4A16 / KV C8 这些方案在 A2 即可用。
- **FP8(E4M3 / E5M2)**:**A3 才谈得上原生**,A2 不要指望。
- **MXFP8 / HiF8 / MXFP4**:**昇腾 950(A5)才原生**。之前量化速查里那句「MX 格式需确认 SOC 版本」,硬件落点就在这里。

## 四、昇腾 950(A5)低精度三个硬实锤

来自白皮书 4.1.1 Cube Core / 4.1.2 Vector Core:

1. **Cube 核原生支持 HiF8 / MXFP8 / FP8 / MXFP4**。同频下,HiF8/MXFP8/FP8 提供 **2× FP16** 张量算力,**MXFP4 提供 4×**;相比上一代 BF16,MXFP4 张量浮点峰值算力提升约 **4 倍**。

2. **HiF8 是这一代看点**(自研 8-bit 浮点,图4-4):
   - 用变长前缀码(点位域 Dot)实现**锥形精度**——绝对值靠近 1 精度高,远离渐降,无跳变。
   - 综合阶码范围达 **[-22, 15] 共 38 个 2 的幂**,接近 FP16 的 40 个。
   - **相比 MXFP8,HiF8 不需要额外的 8 位 MX 缩放因子**——单 8bit 自带足够动态范围,省存储省带宽。
   - 4 个特殊值编码(ZERO/NAN/+INF/-INF),不区分正负 0。

3. **即时量化写回(随路转换)**:Cube 写回 Unified Buffer 阶段可直接做数据格式转换(FP32→BF16/FP16/FP8)与排布转换(NZ→ND/DN);并支持 L0C→UB 随路量化(FP32/INT32 → BF16/FP16/FP8/INT8)。**量化/反量化顺带做,不占额外搬运带宽**——对 W8A8/W4A8 这类在线量化方案是直接利好。

> Vector 核侧格式:全面支持 `FP32 / FP16 / BF16 / INT8 / INT16 / INT32 / INT64`,单核 FP16/FP32 算力较上代 +100%,使 FlashAttention 等 Cube-Vector 融合算子的非矩阵操作不再是短板。

## 五、昇腾 950 算力表(Cube+Vector 总算力)

| 精度 | 950PR | 950DT |
|---|---|---|
| **MXFP4** (TFLOPS) | 1784 / 1561 | 2007 / 1784 / 1561 |
| **HiF8 / MXFP8 / FP8** (TFLOPS) | 919 / 804 | 1034 / 919 / 804 |
| **INT8** (TOPS) | 919 / 804 | 1034 / 919 / 804 |
| **BF16 / FP16** (TFLOPS) | 486 / 425 | 547 / 486 / 425 |
| **TF32** (TFLOPS) | 243 / 212 | 273 / 243 / 212 |

> 同一格多个数字是不同 binning 规格档;950DT 比 950PR 多一档更高规格。规律清晰:**精度每降一档,算力约翻倍**(BF16 → FP8/HiF8/MXFP8 → MXFP4 ≈ 1 : 2 : 4)。

## 六、8-bit / 4-bit 格式位宽对照(白皮书 图4-3)

| 格式 | 符号 | 指数 | 尾数 | 备注 |
|---|:--:|:--:|:--:|---|
| FP8 (E5M2) | 1 | 5 | 2 | 动态范围大、精度低 |
| FP8 (E4M3) | 1 | 4 | 3 | 精度高、范围小 |
| **HiF8** | 1 | 变长(Dynamic) | 变长 | 锥形精度,无需 MX scale |
| FP4 | 1 | 2 | 1 | 配 MX scale → MXFP4 |

## 七、对量化方案落地的指引

- vllm-ascend 里 `W4A4_MXFP4` / `W8A8_MXFP8` 等 scheme,**只有跑在昇腾 950 上才有原生算力**;在 A2/A3 上多为降级路径。
- 纯整数方案(W8A8 / W4A8_DYNAMIC / W4A16 / KV C8)在 A2/A3 即可获得成熟原生支持,是当下生产最稳的选择。
- 选型口诀:**要 MXFP4/HiF8 → 认准 950(A5);要 FP8 → A3 起;只用 INT8 系 → 三代皆可**。
